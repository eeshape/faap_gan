01

Motivation — 교수님 아이디어

교수님 피드백의 핵심: "남성 이미지가 들어올 때와 여성 이미지가 들어올 때, 다른 loss를 적용하면 어떨까?"

교수님 핵심 아이디어

입력 이미지의 성별을 조건으로 삼아, 남성 입력에는 검출 보존만 적용하고 여성 입력에는 공정성 정렬 + 보존을 함께 적용한다. Loss 함수 설계 단계에서 성별 비대칭성을 명시적으로 반영하는 것이 핵심이다.

기존 문제 인식

fix8 (GAN 기반)은 Discriminator가 성별 구분을 못하도록 Generator를 학습하는 방식이다. 이는 남성과 여성 feature를 모두 변형시키는 암묵적 정렬이다. 그러나 이상적으로는 타겟(남성)은 고정하고 소스(여성)만 끌어당기는 방향이 더 자연스럽다.

교수님 직관

"남성은 이미 잘 검출된다 → 건드리지 말고 그냥 보존해라. 여성은 부족하다 → 공정성 손실로 남성 분포에 맞춰라." 이 단순한 직관을 loss 설계에 직접 반영하면 된다.

Loss-level Separation

fix10이 gradient-level 분리(남성 feature를 detach하여 안정성 손실 추가)를 시도했다면, fix11은 더 근본적인 loss-level 분리를 택한다. 남성 샘플에는 contrastive loss 자체를 적용하지 않는다. 신호가 없으면 Generator는 남성 feature를 바꿀 이유가 없다.

핵심 철학

"명시적 패널티(fix10의 stability loss)보다 신호 부재(fix11의 conditional loss)가 더 우아하다." Generator는 역전파 신호가 없으면 변화를 주지 않는다. 이것이 fix11의 핵심 직관이다.

02

Core Concept

배치 내 각 샘플에 대해 성별 조건에 따라 서로 다른 손실을 적용한다. 분기(branching)는 loss 계산 단계에서만 일어나며, Generator와 DETR 구조는 동일하다.

핵심 포인트

성별 분기는 오직 loss 계산 단계에서만 발생한다. Generator, DETR forward pass, feature 추출은 기존과 동일하다. 추가 forward pass가 필요 없으므로 fix10 대비 연산 비용이 낮다.

03

Pipeline — 조건부 분기

전체 학습 파이프라인은 fix8/fix10과 동일하나, Generator 업데이트 단계의 loss 계산에서 성별에 따라 적용되는 항이 달라진다.

Generator Update — fix11 Loss Computation python 
# 1. 배치에서 성별별 인덱스 분리
female_idx = [i for i, g in enumerate(genders) if g == "female"]
male_idx   = [i for i, g in enumerate(genders) if g == "male"]

# 2. Generator forward (전체 배치, 성별 무관)
delta = generator(images)          # G(x) for all samples
perturbed = images + delta

# 3. Frozen DETR forward (전체 배치, 1회)
outputs, features = detr.forward_with_features(perturbed)
# features: (batch, num_queries, 256)

# ─── MALE PATH: Detection only ──────────────────────────
L_det_male = detr.criterion(
    index_select(outputs, male_idx),
    index_select(targets, male_idx)
)
L_male = beta_male * L_det_male          # no contrastive term

# ─── FEMALE PATH: Contrastive + Detection ───────────────
feat_f = features[female_idx]            # female features (anchor)
feat_m = features[male_idx].detach()   # male features (target, detached)

L_contrastive = score_weighted_contrastive(
    feat_f, feat_m, scores_f, scores_m
)
L_det_female = detr.criterion(
    index_select(outputs, female_idx),
    index_select(targets, female_idx)
)
L_female = lambda_con * L_contrastive + beta * L_det_female

# ─── TOTAL ───────────────────────────────────────────────
L_total = L_female + L_male
L_total.backward()
optimizer_G.step()
fix10과의 차이

fix10은 stability_loss를 위해 남성 샘플에도 추가 DETR forward pass가 필요하다. fix11은 남성 샘플에 contrastive loss를 아예 적용하지 않으므로 추가 forward pass 없음. 같은 배치의 features를 성별로 나눠 쓰는 것이 전부다.

04

Male Path — Detection-only Loss

남성 샘플에 대해서는 검출 성능 보존 손실만 적용한다. 공정성 정렬 신호를 받지 않으므로 Generator는 남성 feature를 변형할 이유가 없다.

Male Path ♂
검출 보존만, 공정성 패널티 없음

남성 이미지가 들어오면 Generator는 오직 "원본 DETR의 검출 결과를 유지하라"는 신호만 받는다. 공정성 정렬 방향으로 feature를 끌어당기는 힘이 전혀 없다.

결과적으로 Generator가 만드는 남성 perturbation은 검출에 중립적인 (혹은 검출을 약간 개선하는) 미세한 변형에 머문다.

L_male = β_male · L_det_male
Male Loss Formula
$$\mathcal{L}_{\text{male}} = \beta_{\text{male}} \cdot \mathcal{L}_{\text{det}}\!\left(f_\theta(x_m + G(x_m)),\, y_m\right)$$
$\mathcal{L}_{\text{det}}$: DETR criterion (classification + bbox regression + GIoU)
$\beta_{\text{male}}$: 남성 detection loss 가중치 (fix10의 β와 별도로 조정 가능)
contrastive term 없음: 남성 feature는 공정성 신호를 받지 않는다.
🔒
암묵적 보존

공정성 신호가 없으면 Generator는 남성 feature를 건드릴 이유가 없다. detection loss만으로도 이미지 구조를 크게 바꾸지 않는 방향으로 수렴한다.

⚡
연산 절약

fix10의 stability loss는 남성 샘플의 원본 features가 필요해 추가 DETR forward pass를 요구한다. fix11은 그런 비교 기준이 필요 없으므로 계산 비용이 낮다.

설계 근거

"신호가 없으면 변화도 없다" — Generator의 역전파는 loss에서 오는 gradient에 의해서만 파라미터를 갱신한다. 남성 경로에 공정성 gradient가 없으면 남성 perturbation은 자연스럽게 검출 보존에 최적화된다. 이것이 fix10의 명시적 stability penalty보다 더 근본적인 접근이다.

05

Female Path — Contrastive + Detection

여성 샘플에 대해서는 공정성 정렬(contrastive)과 검출 보존(detection) 두 신호를 함께 적용한다. 여성 feature를 남성 feature 분포 쪽으로 끌어당기면서 검출 성능을 유지한다.

Female Path ♀
공정성 정렬 + 검출 보존

여성 이미지가 들어오면 Generator는 두 가지 목표를 동시에 달성해야 한다: (1) 여성 feature를 남성 feature에 가깝게 만들어 DETR의 성별 편향을 줄이고, (2) 원본 검출 성능을 유지한다.

여성 샘플만이 contrastive loss의 anchor가 된다. 남성 feature는 target(positive)으로 detach되어 사용된다.

L_female = λ_con · L_contrastive + β · L_det_female
Female Loss Formula
$$\mathcal{L}_{\text{female}} = \lambda_{\text{con}} \cdot w(\mathbf{e}) \cdot \mathcal{L}_{\text{SW-Con}}(\mathbf{e}_f, \,\text{sg}[\mathbf{e}_m]) + \beta \cdot \mathcal{L}_{\text{det}}\!\left(f_\theta(x_f + G(x_f)),\, y_f\right)$$
$\mathbf{e}_f$: 여성 DETR decoder features (anchor, gradient 흐름)
$\text{sg}[\mathbf{e}_m]$: 남성 features (stop-gradient / detach, 타겟 역할)
$w(\mathbf{e})$: Score-Weighted Gaussian kernel 가중치
$\lambda_{\text{con}}$: contrastive loss 가중치
$\beta$: detection loss 가중치 (beta scheduling)
Stop-Gradient 설계

남성 feature를 .detach()하여 gradient가 흐르지 않게 한다. 이렇게 하면 contrastive loss가 남성 Generator 파라미터에 영향을 미치지 않는다. 즉, 여성 쪽만 당겨지고 남성은 고정된 타겟이 된다. 이것은 Wasserstein loss의 male_scores.detach()와 같은 철학이다.

🎯
단방향 정렬

여성 feature → 남성 feature 방향으로만 힘이 작용한다. 남성 feature는 detach로 고정된 타겟이므로 당겨지지 않는다. fix8의 양방향 GAN 정렬보다 명확한 방향성을 갖는다.

⚖️
두 신호의 균형

$\lambda_{\text{con}}$과 $\beta$의 비율이 공정성 개선과 검출 보존 사이의 트레이드오프를 결정한다. female path만 이 균형을 조율하므로 실험 변수가 명확하다.

06

Score-Weighted Contrastive Loss

fix10과 동일한 Score-Weighted Contrastive Loss를 사용한다. 단, fix11에서는 여성 샘플만이 anchor가 되며, 남성 positive의 projection은 detach된다.

fix10과 공유되는 설계

fix10에서 설계된 Gaussian kernel 기반 score-weighted contrastive loss를 그대로 사용한다. Bin 방식이 아닌 연속 가중치를 사용하며, $\tau=0.1$의 temperature를 적용한다.

Score-Weighted Contrastive Loss
$$\mathcal{L}_{\text{SW-Con}} = -\frac{1}{|\mathcal{F}|}\sum_{a \in \mathcal{F}} \log \frac{\sum_{p \in \mathcal{M}} w(a,p)\exp\!\left(\frac{\mathbf{z}_a \cdot \text{sg}[\mathbf{z}_p]}{\tau}\right)}{\sum_{p \in \mathcal{M}} w(a,p)\exp\!\left(\frac{\mathbf{z}_a \cdot \text{sg}[\mathbf{z}_p]}{\tau}\right) + \sum_{n \in \mathcal{F} \setminus \{a\}} \exp\!\left(\frac{\mathbf{z}_a \cdot \mathbf{z}_n}{\tau}\right)}$$
$\mathcal{F}$: female 샘플 집합 (anchor)
$\mathcal{M}$: male 샘플 집합 (positive target)
$\text{sg}[\cdot]$: stop-gradient (male projection detached)
$\tau = 0.1$: temperature parameter
Gaussian Kernel Score Weight
$$w(a, p) = \exp\!\left(-\frac{(s_a - s_p)^2}{2\sigma^2}\right)$$
$s_a$: anchor(여성) 샘플의 objectness score
$s_p$: positive(남성) 샘플의 objectness score
$\sigma$: kernel bandwidth (hyperparameter)
직관: 비슷한 objectness score를 가진 쌍에 더 높은 가중치 부여 → 공정한 비교
Score-Weighted Contrastive — fix11 적용 형태 python 
def score_weighted_contrastive(feat_f, feat_m_detached, scores_f, scores_m):
    """
    feat_f: (N_f, num_queries, 256) — female features (anchor)
    feat_m_detached: (N_m, num_queries, 256) — male features (detached, target)
    scores_f, scores_m: objectness scores for weighting
    """
    # Mean-pool over queries
    z_f = F.normalize(feat_f.mean(dim=1), dim=-1)      # (N_f, 256)
    z_m = F.normalize(feat_m_detached.mean(dim=1), dim=-1)  # (N_m, 256)

    # Gaussian kernel weights: w(a, p) = exp(-(s_a - s_p)^2 / 2σ²)
    score_diff = scores_f.unsqueeze(1) - scores_m.unsqueeze(0)  # (N_f, N_m)
    W = torch.exp(-score_diff**2 / (2 * sigma**2))

    # Similarities: female vs male (positive), female vs female (negative)
    sim_pos = (z_f @ z_m.T) / tau       # (N_f, N_m)
    sim_neg = (z_f @ z_f.T) / tau       # (N_f, N_f)

    # Weighted numerator
    num = (W * torch.exp(sim_pos)).sum(dim=1)

    # Denominator: weighted positive + unweighted negative (excl. self)
    mask = 1 - torch.eye(len(z_f), device=z_f.device)
    den  = num + (mask * torch.exp(sim_neg)).sum(dim=1)

    return -torch.log(num / den).mean()

Total Loss — 전체 수식

fix11의 총 Generator 손실은 성별 조건부로 구성된 세 항의 합이다. Discriminator를 완전히 제거했으며, GAN adversarial loss도 없다.

fix11 Total Generator Loss
$$\mathcal{L}_{\text{total}} = \underbrace{\lambda_{\text{con}} \cdot w(\mathbf{e}) \cdot \mathcal{L}_{\text{SW-Con}}(\mathbf{e}_f,\, \text{sg}[\mathbf{e}_m])}_{\text{(1) 공정성 정렬 — female anchor only}} + \underbrace{\beta \cdot \mathcal{L}_{\text{det}}^{\text{female}}}_{\text{(2) 여성 검출 보존}} + \underbrace{\beta_m \cdot \mathcal{L}_{\text{det}}^{\text{male}}}_{\text{(3) 남성 검출 보존}}$$
(1): 여성 feature를 남성 분포로 정렬 — 여성 anchor, 남성 타겟(detached)
(2): 여성 샘플에 대한 DETR criterion — perturbation이 검출을 망치지 않도록
(3): 남성 샘플에 대한 DETR criterion — 남성 성능도 유지
$\beta_m$ vs $\beta$: 남성/여성 detection 가중치를 별도로 설정 가능 (기본값 동일)
왜 항이 3개인가

항 (2)와 (3)을 합쳐서 하나의 $\mathcal{L}_{\text{det}}^{\text{all}}$로 쓸 수도 있다. 하지만 분리하면 여성/남성 detection 가중치를 독립적으로 조정할 수 있다는 유연성이 생긴다. 예컨대 $\beta_m > \beta$로 설정하면 남성 보존을 더 강하게 강제할 수 있다.

Loss 구성 시각화 (예시 가중치)
L_contrastive (여성 공정성) λ_con × w(e)
L_det_female (여성 검출 보존) β
L_det_male (남성 검출 보존) β_m
최종 Loss 조합 코드 python 
# Total Generator Loss (fix11)
L_total = (
    lambda_con * score_weight * L_contrastive  # female anchors only
  + beta       * L_det_female                  # female detection preservation
  + beta_male  * L_det_male                    # male detection preservation
)

# Note: No adversarial loss, no Discriminator
# Note: No Wasserstein loss (unlike fix8/7th)
# Note: beta_male = beta by default, can be tuned separately

fix10 vs fix11 — 트레이드오프

두 접근 모두 Score-Weighted Contrastive Loss와 남성 feature detach를 공유하지만, 남성 샘플을 처리하는 전략에서 근본적으로 다르다.

어떤 것이 더 나은가?

이론적으로 fix11이 더 우아하고 연산 효율적이다. 그러나 fix10의 명시적 stability loss는 남성 feature 변형을 더 강하게 억제할 수 있다는 장점이 있다. 실험적으로 둘을 비교하는 것이 가장 좋은 판단 기준이다. 두 아이디어를 결합 (stability + conditional)하는 것도 선택지다.

✂️
fix11의 강점

"아무것도 하지 않는 것이 최선의 보존이다." Loss 없음 = gradient 없음 = 변화 없음. 이 논리는 직관적이고 설명하기 쉽다.

🔧
fix10의 강점

명시적 패널티는 "얼마나 강하게 보존할 것인가"를 $\gamma_{\text{stability}}$ 하이퍼파라미터로 세밀하게 제어할 수 있다. 디버깅도 더 직접적이다.

Expected Q&A

심사 또는 발표 시 예상되는 질문과 답변.

Q 남성 샘플에 contrastive loss를 아예 안 주면, Generator가 남성 이미지를 완전히 무시하거나 예상치 못한 방향으로 학습하지 않나요? ▾

좋은 지적입니다. 남성 샘플에도 $\beta_m \cdot \mathcal{L}_{\text{det}}^{\text{male}}$ 항이 적용되므로 Generator는 남성 이미지에 대해서도 검출 성능을 보존하는 방향으로 학습됩니다. "무시"가 아니라 "검출에 neutral한 perturbation"을 만들도록 학습됩니다.

만약 detection loss도 없다면 Generator가 남성 이미지에 임의의 perturbation을 줄 수 있지만, fix11에서는 남성 경로에도 detection loss가 살아있어 이 문제를 방지합니다. 이것이 단순히 "남성을 완전히 빼는 것"과 다른 핵심 설계 포인트입니다.

Q fix11과 fix10을 비교했을 때 어떤 경우에 fix11이 더 나을 것으로 예상하나요? ▾

fix11이 유리할 조건은 두 가지입니다. 첫째, 배치 크기가 작을 때입니다. fix10의 stability loss는 남성 원본 features를 저장하고 비교하므로 메모리와 연산이 추가됩니다. fix11은 추가 DETR forward pass가 없으므로 배치당 연산량이 절약됩니다.

둘째, 남성 feature의 안정성이 이미 detection loss만으로 충분히 보장되는 경우입니다. 실험적으로 fix10의 stability loss 가중치 $\gamma$가 낮게 설정되어도 유사 성능이 나온다면, fix11의 설계가 더 간결하고 우아합니다. 최종적으로는 실험 결과로 판단해야 합니다.

Q 배치 내 여성/남성 비율이 불균형하면 contrastive loss 계산에 문제가 생기지 않나요? ▾

좋은 질문입니다. fix11의 contrastive loss는 여성이 anchor, 남성이 positive이므로 한쪽이 0개면 해당 배치에서 contrastive loss를 계산할 수 없습니다. 이를 방지하기 위해 fix8과 동일한 BalancedSubsetSampler를 사용하면 됩니다. 각 배치에 최소 1개 이상의 남성/여성 샘플이 보장되도록 샘플링을 구성합니다.

또한 배치 내 female_idx가 비어있는 엣지 케이스에 대한 방어 코드가 필요합니다. 실제 구현 시 contrastive loss 계산 전에 성별 샘플 수를 체크하고, 한쪽이 없으면 해당 항을 0으로 처리하거나 배치를 재샘플링하는 전략을 사용합니다.

Q fix10 (stability loss)과 fix11 (conditional loss)을 합칠 수는 없나요? 어떤 장단점이 있나요? ▾

두 아이디어를 결합하면 fix11 + stability가 됩니다: 남성에는 contrastive loss 없음 + stability loss 추가. 여성에는 contrastive + detection. 이 경우 남성 feature 보존은 더욱 강하게 보장됩니다.

단점은 fix10과 fix11을 각각 검증하기 전에 결합하면 어느 쪽의 효과인지 분리하기 어렵습니다. ablation study 관점에서는 fix10과 fix11을 먼저 독립적으로 실험한 뒤, 최선의 결합을 탐색하는 것이 올바른 순서입니다. 논문 작성 시에도 "각 아이디어의 독립적 기여"를 보여야 하므로 분리 실험이 필요합니다.
