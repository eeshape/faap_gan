01 · Motivation

왜 남성 분포를 고정해야 하는가

💬
교수님 피드백 (핵심 요약)
"contrastive learning은 남기되, Wasserstein은 제거해라. 그리고 남성 분포가 움직이지 않도록 loss에서 명시적으로 보장해야 한다."

기존 fix8의 Score-Weighted Contrastive Loss는 여성 feature를 남성 feature 방향으로 당기는 역할을 한다. 그런데 이 구조에서 남성 feature가 실제로 고정되는가?

fix8의 한계: detach()만으로는 부족하다

fix8에서는 contrastive loss의 positive target (남성 feature)에 sg(·) = .detach()를 붙인다. 이는 contrastive loss를 통한 gradient가 남성 쪽으로 역전파되지 않도록 차단한다.

하지만 남성 이미지도 Generator를 통과하며 perturbation을 받는다. Detection loss($\mathcal{L}_{det}$) 등 다른 경로를 통해 남성 feature가 여전히 변할 수 있다. detach()는 특정 loss의 gradient path만 선택적으로 차단할 뿐이다.

💡
핵심 통찰: 남성 feature가 모든 경로에서 실제로 변하지 않는다는 것을 보장하려면, feature 자체의 변화량에 직접 패널티를 주는 별도 loss가 필요하다. 이것이 $\mathcal{L}_{\text{stab}}$ 도입의 동기다.

fix10의 한 줄 요약

"여성 feature만 남성 방향으로 이동시키되, 남성 분포는 고정한다."

02 · Overview

3-Component Loss 구조

1 $\mathcal{L}_{\text{con}}$
Contrastive Loss
여성 → 남성 feature 정렬. Gaussian kernel 가중치 + target detach.
2 $\mathcal{L}_{\text{stab}}$
Stability Loss
남성 feature 변화 억제. perturbed vs original MSE. 신규 핵심 요소.
3 $\mathcal{L}_{\text{det}}$
Detection Loss
전체 검출 성능 보존. DETR criterion 원본 그대로.
Total Generator Loss
$$\mathcal{L} = \lambda_{\text{con}} \cdot w(e) \cdot \mathcal{L}_{\text{con}} + \lambda_{\text{stab}} \cdot \mathcal{L}_{\text{stab}} + \beta(e) \cdot \mathcal{L}_{\text{det}}$$
손실 항역할대상 성별스케줄
$\mathcal{L}_{\text{con}}$ 여성 feature를 남성 방향으로 정렬 Female (anchor), Male (target, sg) $w(e)$ epoch 가중치
$\mathcal{L}_{\text{stab}}$ 남성 feature 변화를 명시적 억제 Male only constant $\lambda_{\text{stab}}$
$\mathcal{L}_{\text{det}}$ 전체 검출 성능 보존 All $\beta(e)$ 선형 증가
🗑
Wasserstein 제거 이유: 교수님 지시에 따라 완전 제거. Wasserstein은 score 공간에서의 간접적 분포 정렬로, feature 공간에서 직접 남성 분포를 고정하는 $\mathcal{L}_{\text{stab}}$이 목적에 더 부합하고 해석이 명확하다.
03 · Pipeline

Dual DETR Forward Pass

fix10의 핵심 구조 변화: Stability Loss를 계산하려면 원본(non-perturbed) 이미지에서의 남성 feature가 필요하다. Generator update 시 DETR forward pass가 2회 실행된다.

fix10 · Generator Update Pipeline
Input Batch male + female images
→
Generator $G$ learns $\delta$, $\|\delta\|_\infty \leq \varepsilon$
→
Perturbed Images $x + \delta$ (male & female)
↓ two parallel DETR forwards ↓
Path A — Perturbed grad ON
DETR (frozen params) forward on $x + \delta$
→ $f^{\text{pert}}_F$, $f^{\text{pert}}_M$, pred_logits, pred_boxes
→ $\mathcal{L}_{\text{con}}$ (female anchor, male sg)
→ $\mathcal{L}_{\text{stab}}$ input: $f^{\text{pert}}_M$ (grad flows)
→ $\mathcal{L}_{\text{det}}$ (all outputs)
Path B — Original no_grad
DETR (frozen params) forward on original $x_M$
→ $f^{\text{orig}}_M$ only (male features)
→ $\mathcal{L}_{\text{stab}}$ target: .detach()
→ backward 없음 (frozen reference)
→ 추가 메모리: male feature 텐서만
⚡
구현 포인트: Path B는 torch.no_grad() 블록 안에서 실행되므로 activation이 저장되지 않는다. male sample만 forward하므로 추가 연산 비용은 배치의 ~50% 분량의 DETR forward 1회로 제한된다.
04 · Contrastive Loss

Score-Weighted Contrastive Loss

fix8에서 검증된 구조를 유지. 여성 feature를 anchor, 남성 feature를 positive target으로 하여 feature 공간에서 정렬한다. fix10에서의 변경점은 이산 quality bin을 연속 Gaussian kernel로 교체한 것이다.

Gaussian Kernel 가중치

anchor(female)와 positive(male)의 detection score 차이에 기반하여 연속적인 쌍별 가중치를 부여한다.

Pairwise Weight — Gaussian Kernel
$$w(a, p) = \exp\!\left(-\frac{(s_a - s_p)^2}{2\sigma^2}\right)$$

$s_a$, $s_p$: anchor(female)와 positive(male)의 detection score. Score 차이가 작을수록 $w \to 1$, 클수록 $w \to 0$으로 부드럽게 감소한다. $\sigma$로 가중치 폭을 조절.

Target Detach (sg 연산자)

Asymmetric Similarity — target detached
$$\text{sim}(a, p) = \frac{z_a^\top \cdot \texttt{sg}(z_p)}{\tau}$$

$z_a$, $z_p$: projection head를 거친 L2-normalized feature 벡터. $\tau$: temperature. $\texttt{sg}(\cdot)$: .detach(). 이로써 contrastive loss의 gradient는 여성 feature ($z_a$)에만 흐르고 남성 feature ($z_p$)에는 흐르지 않는다.

Contrastive Loss 공식

L_contrastive
$$\mathcal{L}_{\text{con}} = -\!\sum_{(a,p)} w(a,p) \cdot \log \frac{\exp(\text{sim}(a,p))}{\exp(\text{sim}(a,p)) + \displaystyle\sum_{n \in \mathcal{N}} \exp(\text{sim}(a,n))}$$

$\mathcal{N}$: negative 집합 (same-gender, 여성-여성 쌍). 이 loss는 여성 feature에만 gradient를 생성하며, 남성 feature는 fixed target으로만 사용된다.

python# Score-Weighted Contrastive Loss — fix10 def score_weighted_contrastive(z_f, z_m, s_f, s_m, tau=0.07, sigma=0.3): # z_f: (N_f, D), z_m: (N_m, D) — L2 normalized # s_f: (N_f,), s_m: (N_m,) — detection scores # Stop gradient on male features (contrastive path only) z_m_sg = z_m.detach() # (N_m, D) # Gaussian kernel: weight for each (female_i, male_j) pair s_diff_sq = (s_f.unsqueeze(1) - s_m.unsqueeze(0)) ** 2 # (N_f, N_m) weights = torch.exp(-s_diff_sq / (2 * sigma ** 2)) # (N_f, N_m) # Positive similarity: each female paired with closest male sim_pos = (z_f * z_m_sg.mean(0, keepdim=True)).sum(-1) / tau # (N_f,) # Negative similarity: female-female (intra-gender) sim_neg = (z_f @ z_f.T) / tau # (N_f, N_f) sim_neg.fill_diagonal_(-float('inf')) # exclude self logits = torch.cat([sim_pos.unsqueeze(1), sim_neg], dim=1) # (N_f, 1+N_f) labels = torch.zeros(len(z_f), dtype=torch.long, device=z_f.device) loss = F.cross_entropy(logits, labels, reduction='none') # (N_f,) w_per_anchor = weights.mean(dim=1) # (N_f,) return (w_per_anchor * loss).mean()
📌
fix8 대비 변경점: 이산 quality bin (상/중/하) → 연속 Gaussian kernel. 경계에서의 gradient 불연속 문제 해소. $\sigma$ 단일 파라미터로 가중치 특성 제어 가능.
05 · Stability Loss

Male Feature Stability Loss

fix10의 핵심 신규 요소

남성 feature가 perturbation에 의해 변하지 않도록 명시적 패널티를 부여한다. detach()로 차단할 수 없는 모든 gradient 경로를 통한 변화까지 억제한다.

직관적 이해

Generator가 남성 이미지에 perturbation $\delta_M$을 추가하면 DETR이 추출하는 남성 feature가 미묘하게 달라진다. 이 변화가 작더라도 contrastive loss의 "이동 타겟"이 흔들리는 문제가 생긴다. Stability Loss는 이 현상을 직접 패널티로 억제한다.

L_stability — Male Feature MSE
$$\mathcal{L}_{\text{stab}} = \text{MSE}\!\left(f^{\text{pert}}_M,\; \texttt{sg}\!\left(f^{\text{orig}}_M\right)\right) = \frac{1}{N_M \cdot D} \sum_{i=1}^{N_M} \left\| f^{\text{pert}}_{M,i} - \texttt{sg}\!\left(f^{\text{orig}}_{M,i}\right) \right\|_2^2$$

$f^{\text{pert}}_M$: perturbed 남성 이미지의 DETR feature (gradient 있음 → Generator로 역전파). $f^{\text{orig}}_M$: 원본 남성 이미지의 DETR feature (torch.no_grad() + .detach()로 완전 차단).

구현: 추가 Forward Pass

python · generator update# ── Step 1: Get frozen male reference features (no gradient) ────── with torch.no_grad(): _, f_orig_m = detr.forward_with_features(original_male_images) f_orig_m = f_orig_m.detach() # (N_m, Q, D) — reference # ── Step 2: Generate perturbations ──────────────────────────────── delta_m = generator(male_images) # (N_m, C, H, W) delta_f = generator(female_images) # (N_f, C, H, W) perturbed_m = (male_images + delta_m).clamp(0, 1) perturbed_f = (female_images + delta_f).clamp(0, 1) # ── Step 3: DETR forward on perturbed (grad ON) ─────────────────── outputs_m, f_pert_m = detr.forward_with_features(perturbed_m) outputs_f, f_pert_f = detr.forward_with_features(perturbed_f) # ── Step 4: Compute losses ──────────────────────────────────────── L_stab = F.mse_loss(f_pert_m, f_orig_m) # grad flows to G via f_pert_m L_con = score_weighted_contrastive( z_f=proj(f_pert_f), z_m=proj(f_pert_m), s_f=scores_f, s_m=scores_m ) outputs_all = merge_outputs(outputs_m, outputs_f) L_det, _ = detr.criterion(outputs_all, targets_all) # ── Step 5: Total loss & backward ───────────────────────────────── L = lambda_con * w_e * L_con + lambda_stab * L_stab + beta_e * L_det optimizer_g.zero_grad() L.backward() clip_grad_norm_(generator.parameters(), max_norm=0.1) optimizer_g.step()

Gradient 경로

$\mathcal{L}_{\text{stab}}$에서 gradient가 흐르는 경로:

$\mathcal{L}_{\text{stab}} = \text{MSE}(f^{\text{pert}}_M,\; f^{\text{orig}}_M)$
  ↓ ∂L / ∂f^{pert}_M (gradient flows HERE)
  ↓ DETR backbone (frozen: no param update, but passes gradients)
  ↓ ∂f^{pert}_M / ∂δ_M
  ↓ Generator $G$ — parameters updated
  × $f^{\text{orig}}_M$ side: no_grad + detach → zero gradient

🔑
왜 MSE인가? feature vector의 방향과 크기를 모두 유지한다. Cosine similarity 기반 loss는 방향만 보존하므로 feature scale이 달라질 수 있다. 남성 분포를 진정으로 고정하려면 MSE가 더 강한 제약이다.
06 · Detection Loss

Detection Loss (변경 없음)

fix8과 동일하게 DETR 원본 criterion을 사용한다. Perturbation 학습 중 검출 성능이 저하되지 않도록 anchor 역할을 한다.

L_det — DETR Criterion
$$\mathcal{L}_{\text{det}} = \mathcal{L}_{\text{cls}} + \mathcal{L}_{\text{bbox}} + \mathcal{L}_{\text{giou}}$$

Beta 스케줄링 (선형 증가)

학습 초반에는 fairness 개선에 집중하고, 후반으로 갈수록 detection 성능 보존 비중을 높인다.

Beta Schedule
$$\beta(e) = \beta_{\text{start}} + (\beta_{\text{end}} - \beta_{\text{start}}) \cdot \frac{e}{E - 1}, \quad \beta_{\text{start}} = 0.5,\; \beta_{\text{end}} = 0.6$$

10 · Q&A

예상 질문 답변

Q1 Original forward pass를 추가하면 학습 속도가 많이 느려지지 않나요? ▼

실질적인 overhead는 생각보다 작다. Original forward는 torch.no_grad() 안에서 실행되므로 activation이 저장되지 않는다. DETR forward 계산만 추가되며 backward가 없다.

또한 남성 이미지의 feature만 필요하므로, 배치에서 male sample만 골라 forward하면 된다. 전체 학습 시간 증가는 약 20~30% 수준으로 예상되며, 이는 Wasserstein의 1D 정렬 + 보간 연산을 제거함으로써 일부 상쇄된다.

Q2 $\mathcal{L}_{\text{stab}}$이 너무 강하면 Generator가 남성에 아무 perturbation도 주지 않게 되지 않나요? ▼

정확한 지적이다. $\lambda_{\text{stab}}$이 지나치게 크면 Generator는 남성 입력에 대해 $\delta_M \approx 0$으로 수렴하려 한다. 이는 stability 목적에는 부합하지만 detection loss $\mathcal{L}_{\text{det}}$와 상충이 생길 수 있다.

실용적으로는: (1) 초기 $\lambda_{\text{stab}} = 0.3$으로 시작, (2) $\mathcal{L}_{\text{stab}}$ 값이 빠르게 $10^{-3}$ 이하로 수렴하면 줄이고, 수렴이 느리면 늘리는 방식으로 조정한다. $\mathcal{L}_{\text{det}}$와 균형을 모니터링하는 것이 핵심이다.

Q3 detach()만으로 충분하지 않다고 했는데, 구체적으로 어떤 경로로 남성 feature가 변할 수 있나요? ▼

$\mathcal{L}_{\text{det}}$를 통한 경로가 대표적이다. Detection loss는 male + female 배치 전체에 대해 계산되며, Generator가 male 이미지에 perturbation을 가하고 그 결과의 detection loss가 역전파된다. 이 경로에는 detach()가 적용되지 않는다.

$\mathcal{L}_{\text{stab}}$은 이러한 모든 경로의 결과를 한꺼번에 억제한다. 경로가 얼마나 많든 결국 feature의 변화량에 직접 패널티를 주기 때문이다. "어떤 경로를 통해 변했는가"와 무관하게, 변화 자체를 측정하여 억제한다.

Q4 Gaussian kernel의 $\sigma$는 어떻게 설정하고, 결과에 얼마나 민감한가요? ▼

Detection score는 0~1 범위이며, $\sigma = 0.3$으로 설정하면 score 차이 0.3 이내의 쌍은 weight $\geq e^{-0.5} \approx 0.6$을 갖고, 차이 0.6 이상은 $\leq 0.1$로 거의 무시된다.

실험적으로 $\sigma \in [0.2, 0.5]$ 범위에서 결과가 크게 달라지지 않는다. 너무 작으면 ($\sigma < 0.1$) 대부분의 쌍이 무시되어 loss가 sparse해지고, 너무 크면 ($\sigma > 0.8$) 모든 쌍이 동등하게 취급되어 uniform weight와 유사해진다. 초기값 $\sigma = 0.3$을 권장하며, 이후 ablation으로 정밀 조정한다.
