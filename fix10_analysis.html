04 — Core Innovation

Score-Based Gaussian Weight

fix8의 이산 quality bin을 연속 Gaussian kernel로 대체합니다. 이것이 fix10의 핵심 변경입니다.

fix8 (이산 bin)
$$\ell_i = \text{bucketize}(s_i,\;[b_1, b_2])$$ $$\text{positive} \iff \ell_a = \ell_p$$
  • score를 3개 bin으로 잘라냄
  • bin 내 = 동일 weight (1.0)
  • bin 밖 = weight 0
  • 경계에서 정보 손실
→
fix10 (연속 kernel)
$$w(a, p) = \exp\!\left(-\frac{|s_a - s_p|^2}{2\sigma^2}\right)$$
  • score 차이를 연속 가중치로 반영
  • 유사할수록 w → 1 (강한 positive)
  • 다를수록 w → 0 (약한 positive)
  • 경계 문제 완전 제거

$\sigma$ (bandwidth) 설정

Adaptive bandwidth
$$\sigma = \text{std}(s_{\text{batch}}) \times \text{bandwidth\_scale}$$

배치 내 score의 표준편차를 기반으로 자동 설정됩니다. bandwidth_scale (기본값 1.0)로 커널의 폭을 조절합니다.

bandwidth_scale$\sigma$ 크기효과
0.5좁음score가 매우 비슷한 pair만 strong positive → 엄격한 매칭
1.0 (기본)표준score 분포에 맞는 자연스러운 폭
2.0넓음더 많은 pair가 positive로 작용 → 관대한 매칭

직관적 예시

# batch scores (5개 이미지) scores = [0.8(남), 0.7(여), 0.5(남), 0.3(여), 0.4(남)] σ = std(scores) ≈ 0.19 # anchor=여(0.7)에 대한 cross-gender weight w(0.7, 0.8남) = exp(-0.01/0.072) = 0.87 ← 강한 positive w(0.7, 0.5남) = exp(-0.04/0.072) = 0.57 ← 중간 positive w(0.7, 0.4남) = exp(-0.09/0.072) = 0.29 ← 약한 positive # fix8이었다면: bin=[0.8,0.7]=bin2, [0.5,0.4]=bin1, [0.3]=bin0 # → 0.7여와 0.8남만 positive (w=1), 나머지 전부 w=0 (정보 손실!)
05 — Score-Weighted Contrastive Loss

Loss 수식

집합 정의

Anchor & Target 집합
$$\mathcal{A} = \{i \mid \text{gender}_i = \text{anchor}\}, \quad \mathcal{T} = \{j \mid \text{gender}_j = \text{target}\}$$

Similarity

Temperature-scaled cosine similarity
$$\text{sim}(i,j) = \frac{z_i^\top z_j}{\tau}, \quad \tau = 0.1$$

Score-Weighted Positive Similarity

fix8의 이산 positive mask를 연속 가중 평균으로 대체합니다.

Weighted positive + stop-gradient (핵심 변경)
$$\overline{\text{sim}}_a^{+} = \frac{\displaystyle\sum_{p \in \mathcal{T}} w(a, p) \cdot \frac{z_a^\top \,\text{sg}(z_p)}{\tau}}{\displaystyle\sum_{p \in \mathcal{T}} w(a, p)}$$

여기서 $w(a,p) = \exp(-|s_a - s_p|^2 / 2\sigma^2)$이고, $\text{sg}(\cdot)$은 stop-gradient (detach)입니다. Score가 유사한 cross-gender pair일수록 더 높은 가중치를 받으며, target(남성) projection은 고정되어 gradient가 anchor(여성)에만 흐릅니다.

💡
fix8 대비 2가지 핵심 차이:
(1) fix8은 $\text{pos\_mask}$가 0 또는 1 (이산 bin). fix10은 $w(a,p)$가 0~1 연속값 (Gaussian kernel).
(2) fix8은 target에 gradient가 흐름 (양방향). fix10은 $\text{sg}(z_p)$로 target 고정 (단방향).

Denominator (Collapse 방지)

LogSumExp over all (self 제외)
$$\text{logdenom}_a = \log \sum_{\substack{k=1 \\ k \neq a}}^{N} \exp\!\big(\text{sim}(a, k)\big)$$

최종 Loss

Score-Weighted Contrastive Loss
$$\boxed{\mathcal{L}_{\text{contrastive}} = \frac{1}{|\mathcal{A}^+|} \sum_{a \in \mathcal{A}^+} \left[-\;\overline{\text{sim}}_a^{+} \;+\; \text{logdenom}_a\right]}$$

$\mathcal{A}^+ = \{a \in \mathcal{A} \mid \sum_p w(a,p) > \epsilon\}$: 유의미한 positive가 있는 anchor만 포함.

코드 대응

Python # 1) Score-based Gaussian weight (bin 대체) sigma = scores.std().clamp(min=0.01) * self.bandwidth_scale score_diff_sq = (scores[anchor_idx].unsqueeze(1) - scores[target_idx].unsqueeze(0)).pow(2) weights = (-score_diff_sq / (2 * sigma**2)).exp() # (n_a, n_t) # 2) 단방향 similarity (target detach — 핵심!) anchor_proj = projections[anchor_idx] # gradient ✓ target_proj = projections[target_idx].detach() # gradient ✗ (고정) sim_a2t = torch.mm(anchor_proj, target_proj.T) / τ # (n_a, n_t) # 3) Weighted positive similarity w_sum = weights.sum(dim=1).clamp(min=1e-8) weighted_pos = (sim_a2t * weights).sum(1) / w_sum # 가중 평균 # 4) Denominator (self 제외, 전체 sim 사용) sim_all = projections @ projections.T / τ sim_anchors = sim_all[anchor_idx].masked_fill(mask_self, -inf) log_denom = torch.logsumexp(sim_anchors, dim=1) # 4) Loss loss = -(weighted_pos - log_denom).mean()
05.5 — Unidirectional Gradient

단방향 Gradient: target detach

7th.py(논문)의 Wasserstein loss는 male_scores.detach()로 남성을 "고정된 목표"로 설정했습니다. fix10에서 Wasserstein을 제거했으므로, contrastive loss 안에서 동일한 단방향 메커니즘을 구현해야 합니다.

문제: detach 없으면 양방향 이동

# detach 없이 similarity 계산하면: sim(a, p) = z_aᵀ · z_p / τ # backprop 시 gradient가 양쪽에 흐름 z_a (여성) ← gradient ✓ # 여성이 이동 z_p (남성) ← gradient ✓ # 남성도 이동 (의도하지 않음!) # 결과: 둘 다 "가운데"로 모임 → 남성 성능 하락 위험

해결: target projection을 detach

단방향 Similarity (stop-gradient)
$$\text{sim}(a, p) = \frac{z_a^\top \cdot \text{sg}(z_p)}{\tau}$$

$\text{sg}(\cdot)$ = stop-gradient. $z_p$의 값은 사용하지만 gradient는 차단합니다.

# fix10: target을 detach하여 단방향 gradient anchor_proj = projections[anchor_idx] # 여성: gradient ✓ target_proj = projections[target_idx].detach() # 남성: gradient ✗ (고정) sim_a2t = torch.mm(anchor_proj, target_proj.T) / τ # 단방향 similarity

Gradient 흐름 비교

detach 없음 (양방향)
여성: x→G→δ→DETR→z_a ← grad ✓ ↕ sim 남성: x→G→δ→DETR→z_p ← grad ✓ → 둘 다 이동 (남성 성능 하락)
→
detach 있음 (단방향)
여성: x→G→δ→DETR→z_a ← grad ✓ ↕ sim 남성: x→G→δ→DETR→z_p.detach() ✗ → 여성만 이동 (남성 고정)
🔒
7th.py와의 연결: 7th.py의 male_scores.detach()가 score 공간에서 남성을 고정했던 것처럼, fix10의 projections[target_idx].detach()는 feature 공간에서 남성을 고정합니다. Wasserstein을 제거했지만 단방향 정렬의 핵심 메커니즘은 계승됩니다.

기존 W거리의 역할fix10에서의 대체 메커니즘
"같은 수준의 score끼리 맞춰라" Gaussian kernel $w(a,p)$가 score 유사도를 직접 반영
Score 분포 정렬 Feature가 정렬되면 DETR 출력(score)도 자연 정렬
단방향 push (anchor ↑) Anchor 방향 설정으로 동일 효과
Per-sample gradient Contrastive weight가 각 pair에 개별 gradient 전달

핵심 논거

fix8에서 W거리는 "score가 비슷한 남녀를 맞춰라"고 말하고, SupCon은 "같은 quality bin의 남녀 feature를 맞춰라"고 말합니다. Quality bin이 score로 정의되므로, 이 두 말은 사실상 동일합니다.

fix10의 Gaussian kernel은 이 두 역할을 하나로 흡수합니다: $w(a,p) = \exp(-|s_a - s_p|^2 / 2\sigma^2)$ 자체가 "score가 비슷한 cross-gender pair를 더 강하게 정렬하라"는 의미이기 때문입니다.

11 — Expected Q&A

예상 질문

Q1. W거리를 빼면 score 정렬이 안 되는 거 아닌가?

Gaussian kernel weight $w(a,p)$가 score 유사도를 직접 반영하므로, contrastive loss 자체가 "score가 비슷한 pair를 정렬하라"는 의미를 내포합니다. Feature가 정렬되면 DETR 출력(score)도 자연 정렬됩니다.

Q2. Gaussian kernel 대신 bin을 유지하면 안 되나?

가능하지만, bin은 경계에서 정보 손실이 발생합니다. score 0.49와 0.51이 다른 bin에 배정되면 negative 관계가 되지만, 실제로는 거의 같은 품질입니다. Gaussian kernel은 이 문제를 자연스럽게 해결합니다.

Q3. $\sigma$를 adaptive로 설정하면 학습이 불안정하지 않나?

배치 크기가 5로 작아서 우려할 수 있지만, $\text{std}(scores)$는 배치 간 크게 변하지 않습니다 (detection score가 대부분 0.3~0.9 범위). 불안정하면 전체 데이터셋의 score std를 사전 계산하여 고정값으로 사용할 수 있습니다.

Q4. 이 연구의 논문 기여는?

(1) 기존 SupCon+Wasserstein의 중복 문제를 지적
(2) Score-weighted contrastive로 단일 목적의 통합 loss 제안
(3) 하이퍼파라미터 감소에도 동등/우수 성능 달성 → Occam's razor
(4) 이산 bin vs 연속 kernel의 실험적 비교 제공

Q5. fix8 대비 성능이 안 나오면?

"단순화에도 성능 유지"도 의미 있는 결과입니다. 하이퍼파라미터 2개 감소 + loss 목적 명확화는 그 자체로 contribution입니다. 만약 성능이 하락한다면, Wasserstein이 contrastive와 독립적인 보완 역할을 한다는 발견이 됩니다.
