Background & Motivation

기존 논문 버전(7th.py)은 GAN 방식으로 공정성을 달성합니다. Generator가 perturbation을 생성하고, Discriminator가 DETR feature에서 성별을 분류하는 적대적 학습 구조입니다.

fix8(이 코드)은 GAN의 Discriminator를 제거하고 Supervised Contrastive Learning(SupCon)으로 대체한 실험입니다.

03

Projection Head

DETR decoder features를 contrastive learning에 적합한 임베딩 공간으로 변환하는 모듈입니다.

Step 1 — Mean Pooling
$$\bar{f}_i = \frac{1}{Q}\sum_{q=1}^{Q} f_i^{(q)} \in \mathbb{R}^{256}$$

여기서 $f_i \in \mathbb{R}^{Q \times 256}$ 은 이미지 $i$의 DETR decoder features이며, $Q=100$은 DETR의 object query 개수입니다. 100개의 query 벡터를 평균 내어 이미지 하나를 대표하는 256차원 벡터로 만듭니다.

Step 2 — 2-Layer MLP
$$h(\bar{f}_i) = W_2 \cdot \text{ReLU}(W_1 \bar{f}_i + b_1) + b_2 \in \mathbb{R}^{128}$$

256차원 → 256차원 (hidden) → 128차원 (output)의 2-layer MLP를 통과합니다. 중간에 ReLU 활성화와 Dropout(p=0.1)이 적용됩니다.

Step 3 — L2 Normalization
$$z_i = \frac{h(\bar{f}_i)}{\|h(\bar{f}_i)\|_2} \in \mathbb{R}^{128}, \quad \|z_i\| = 1$$
🔎
L2 정규화를 하는 이유: $\|z_i\|=1$이면 두 벡터의 내적이 곧 cosine similarity가 됩니다.
$z_i^\top z_j = \|z_i\|\|z_j\|\cos\theta_{ij} = \cos\theta_{ij}$
따라서 별도의 cosine 계산 없이 행렬 곱만으로 모든 쌍의 유사도를 한 번에 구할 수 있습니다.

04

Quality Bin 할당

배치 내 각 이미지에 "검출 품질 등급"을 부여합니다. 이 등급이 contrastive loss에서 positive/negative를 결정하는 기준이 됩니다.

Detection Score 계산

각 이미지의 detection 품질을 하나의 스칼라 score로 요약합니다.

Image-level Detection Score (기본 방법)
$$s_i = \frac{1}{\min(K, Q)} \sum_{q \in \text{top-}K} \max_c \; p_{i,q,c}$$
  • $p_{i,q,c}$: 이미지 $i$, query $q$, 클래스 $c$의 softmax 확률
  • 각 query에서 가장 높은 클래스 확률을 취함 (no-object 클래스 제외)
  • 상위 $K=10$개 query의 평균이 해당 이미지의 detection score

Bin 분할

배치 내 $N$개 이미지의 score를 $K=3$개의 등급으로 분할합니다.

Quantile 기반 경계값
$$b_k = Q\!\left(s,\; \frac{k}{K}\right), \quad k = 1, \ldots, K\!-\!1$$
Bin 라벨 할당
$$\ell_i = \text{bucketize}(s_i,\; [b_1, b_2]) \;\in\; \{0, 1, 2\}$$

$Q(s, p)$는 score 분포의 $p$-분위수입니다. $K=3$이면 경계값이 33%, 67% 분위수에서 잡혀 배치를 3등분합니다.

예시 (batch=5)

# scores = [0.8, 0.7, 0.5, 0.3, 0.4] # quantile boundaries: b₁=0.4, b₂=0.7 이미지0 (남, score=0.8) → bin 2 (상위) 이미지1 (여, score=0.7) → bin 2 (상위) ← 이미지0과 같은 bin! 이미지2 (남, score=0.5) → bin 1 (중위) 이미지3 (여, score=0.3) → bin 0 (하위) 이미지4 (남, score=0.4) → bin 1 (중위) ← 이미지3과 다른 bin

의미: 이미지0(남)과 이미지1(여)은 같은 bin 2 → 서로 positive 관계. 이미지0(남)과 이미지3(여)은 다른 bin → negative 관계.

05

Gender-Centered SupCon Loss

이 코드의 핵심 손실 함수입니다. 특정 성별만 anchor로 지정하여 contrastive gradient의 방향을 제어합니다.

집합 정의

배치 내 인덱스를 성별에 따라 분리합니다.

Anchor & Target 집합
$$\mathcal{A} = \{i \mid \text{gender}_i = \text{anchor\_gender}\}$$ $$\mathcal{T} = \{j \mid \text{gender}_j = \text{target\_gender}\}$$

각 anchor $a \in \mathcal{A}$에 대한 positive 집합을 정의합니다:

Positive 집합 — 반대 성별 & 같은 quality bin
$$\mathcal{P}(a) = \{j \in \mathcal{T} \mid \ell_j = \ell_a\}$$

Similarity 계산

Temperature-scaled Cosine Similarity
$$\text{sim}(i, j) = \frac{z_i^\top z_j}{\tau}, \quad \tau = 0.1$$

L2 정규화된 벡터의 내적이므로 $z_i^\top z_j = \cos\theta_{ij}$ 이고, $\tau=0.1$로 나누면 유사도 값이 10배 증폭됩니다.

$\tau$$\cos\theta=0.9$ 일 때$\cos\theta=0.1$ 일 때차이 배율
1.00.90.1×1
0.1 (사용값)9.01.0×10
0.0190.010.0×100

$\tau$가 작을수록 softmax가 가장 가까운 샘플에 집중하게 되어 hard contrastive learning이 됩니다. 0.1은 SupCon 논문의 표준값입니다.

Loss 수식

Gender-Centered SupCon Loss (최종)
$$\mathcal{L}_{\text{SupCon}} = \frac{1}{|\mathcal{A}^+|} \sum_{a \in \mathcal{A}^+} \left[\; \underbrace{-\frac{1}{|\mathcal{P}(a)|} \sum_{p \in \mathcal{P}(a)} \text{sim}(a, p)}_{\text{(A) positive 유사도 평균}} \;+\; \underbrace{\log \sum_{\substack{k=1 \\ k \neq a}}^{N} \exp\!\big(\text{sim}(a, k)\big)}_{\text{(B) 전체 logsumexp}} \;\right]$$

여기서 $\mathcal{A}^+ = \{a \in \mathcal{A} \mid |\mathcal{P}(a)| > 0\}$ 는 positive가 1개 이상 존재하는 anchor만 포함합니다.

각 항의 의미

(A) 분자 항

Positive 유사도 평균

$$-\frac{1}{|\mathcal{P}(a)|} \sum_{p \in \mathcal{P}(a)} \frac{z_a^\top z_p}{\tau}$$

anchor $a$와 positive(같은 bin, 반대 성별) 사이의 유사도 평균에 음수를 붙인 것.

  • 유사도가 높으면(positive와 가까우면) → 이 항이 작아짐 → loss 감소
  • 유사도가 낮으면(positive와 멀면) → 이 항이 커짐 → loss 증가
(B) 분모 항

전체 LogSumExp

$$\log \sum_{k \neq a} \exp\!\left(\frac{z_a^\top z_k}{\tau}\right)$$

anchor $a$와 자기 자신을 제외한 모든 샘플의 유사도에 대한 logsumexp.

  • 모든 것과 가까우면 → 이 항이 커짐 → loss 증가 (collapse 방지)
  • 이 항이 있어야 "전부 같게 만들기"를 방지

🎯 최적화 방향 요약

(A) 최소화: positive(같은 품질 등급, 반대 성별)와 가깝게 → (B) 최소화: 하지만 모든 것과 가까워지면 안 됨 → 결과적으로 positive만 가깝고 negative는 멀어지는 방향으로 학습

Anchor가 Gradient를 받는다는 것의 의미

⚠️
SupCon loss에서 anchor 쪽의 projection에 gradient가 흐릅니다. Generator는 모든 이미지에 perturbation을 적용하지만, contrastive loss의 gradient가 anchor에서 시작되므로 anchor 성별의 feature 변화에 더 강하게 기여합니다.

anchor_man: gradient가 남성 feature 쪽 → "남성이 움직여서 여성을 닮아라"
anchor_women: gradient가 여성 feature 쪽 → "여성이 움직여서 남성을 닮아라"

Wasserstein Loss (단방향)

Feature 공간의 정렬(SupCon)과 별도로, detection score 분포 자체를 정렬하는 손실입니다.

1D Wasserstein Distance 개념

두 1차원 분포 사이의 거리를 측정합니다. 직관적으로 "한 분포를 다른 분포로 옮기는 데 필요한 최소 이동량"입니다.

남성 scores (정렬): [0.8, 0.6, 0.4] 여성 scores (정렬): [0.6, 0.5, 0.3] 차이: [0.2, 0.1, 0.1] Wasserstein = 평균 차이 = 0.133

정렬된 quantile끼리 1:1 대응시켜 차이의 평균을 구합니다.

단방향(Unidirectional) 설계

anchor_women (여성 → 남성 수준으로 올림, 7th.py와 동일)
$$\mathcal{L}_{\text{wass}} = \frac{1}{K} \sum_{k=1}^{K} \text{ReLU}\!\left(\;\underbrace{\tilde{s}_m^{(k)}}_{\text{남성 (detach)}} - \underbrace{\tilde{s}_f^{(k)}}_{\text{여성 (gradient)}}\;\right)$$
anchor_man (남성 → 여성 수준으로 올림, 역방향)
$$\mathcal{L}_{\text{wass}} = \frac{1}{K} \sum_{k=1}^{K} \text{ReLU}\!\left(\;\underbrace{\tilde{s}_f^{(k)}}_{\text{여성 (detach)}} - \underbrace{\tilde{s}_m^{(k)}}_{\text{남성 (gradient)}}\;\right)$$

여기서 $\tilde{s}$는 크기가 다를 수 있는 두 분포를 $K = \max(|\mathcal{F}|, |\mathcal{M}|)$개로 선형 보간하여 리사이즈한 값입니다.

ReLU의 역할 — "단방향"의 핵심

$$\text{ReLU}(x) = \max(0, x) = \begin{cases} x & x > 0 \\ 0 & x \leq 0 \end{cases}$$

예시 (anchor_women 기준)

# Case 1: 여성이 남성보다 낮음 → 패널티 발생 남성 (detach): [0.8, 0.6, 0.4] 여성: [0.6, 0.5, 0.3] 차이: [0.2, 0.1, 0.1] ← 모두 양수 → ReLU 통과 Loss = 0.133 # 여성을 끌어올려야 함 # Case 2: 여성이 일부 구간에서 남성보다 높음 → 해당 구간은 무시 남성 (detach): [0.8, 0.6, 0.4] 여성: [0.9, 0.5, 0.3] 차이: [-0.1, 0.1, 0.1] ← 첫 번째 음수 → ReLU = 0 Loss = 0.067 # 이미 높은 부분은 패널티 없음

Scheduling 수식

Epsilon 스케줄 (Perturbation 크기)

3단계로 perturbation 크기를 조절합니다.

$$\epsilon(e) = \begin{cases} \epsilon_s + (\epsilon_p - \epsilon_s) \cdot \dfrac{e}{E_w} & e < E_w \quad \text{(Warmup)} \\[8pt] \epsilon_p & E_w \leq e < E_w + E_h \quad \text{(Hold)} \\[8pt] \epsilon_p + (\epsilon_m - \epsilon_p) \cdot \dfrac{e - E_w - E_h}{E_c} & e \geq E_w + E_h \quad \text{(Cooldown)} \end{cases}$$

Beta 스케줄 (Detection Loss 가중치)

선형 증가
$$\beta(e) = \beta_s + (\beta_f - \beta_s) \cdot \frac{e}{E - 1}$$

$\beta_s = 0.5$, $\beta_f = 0.6$. 초반엔 공정성 학습에 집중하고, 후반엔 detection 보존 비중을 높입니다.

Contrastive Warmup

선형 Warmup
$$w(e) = \min\!\left(\frac{e}{E_{\text{cw}}},\; 1.0\right), \quad E_{\text{cw}} = 3$$

왜 Warmup이 필요한가?

학습 초기에는 ProjectionHead의 가중치가 무작위(random)입니다. 이 상태에서 SupCon loss를 바로 켜면 의미 없는 방향으로 gradient가 흘러 학습이 불안정해집니다. 3 epoch 동안 서서히 활성화하여 ProjectionHead가 어느 정도 유의미한 임베딩을 만든 뒤 본격적으로 contrastive 학습을 시작합니다.

Learning Rate 스케줄

$$\text{lr\_scale}(e) = \begin{cases} \dfrac{e+1}{E_w} & e < E_w \quad \text{(Warmup)} \\[8pt] 0.1 + 0.45 \cdot \left(1 + \cos\!\left(\pi \cdot \dfrac{e - E_w}{E - E_w}\right)\right) & e \geq E_w \quad \text{(Cosine)} \end{cases}$$

$E_w = 3$. Warmup 이후 cosine annealing으로 학습률이 1.0에서 약 0.1까지 감소합니다.

예상 Q&A

Q1. fix7이랑 뭐가 다른가?

fix7은 성별 무관하게 모든 샘플이 anchor입니다 (Quality SupCon). fix8은 한 성별만 anchor로 지정하여 contrastive gradient의 방향을 제어하는 실험입니다.

Q2. Anchor 방향이 결과에 어떤 영향을 줄 것으로 예상하는가?

anchor_women(여성→남성)은 7th.py와 동일한 "여성 끌어올리기" 방향이라 유사한 결과가 예상됩니다. anchor_man(남성→여성)은 역방향이라 남성 성능이 약간 하락하면서 gap이 줄어드는 패턴이 예상됩니다.

Q3. Bin을 3개로 나눈 근거는?

batch_size=5에서 너무 많은 bin은 bin당 샘플 수가 너무 적어집니다. 코드에 n_bins = min(3, N//2) 안전장치가 있어 배치 크기에 따라 자동 조절됩니다. Bin 개수에 대한 ablation은 추후 진행 가능합니다.

Q4. GAN(7th) 대비 SupCon(fix8)의 장단점은?

장점: Discriminator가 없어 학습 안정성 향상, D/G 균형 문제 없음, 하이퍼파라미터 감소.
단점: GAN은 feature 수준에서 성별 정보를 직접 제거하지만, SupCon은 "같은 품질이면 비슷하게"라는 간접적 정렬이므로 성별 정보 제거에 대한 이론적 보장이 약합니다.

Q5. Temperature 0.1의 근거는?

SupCon 원본 논문(Khosla et al., 2020)의 표준 설정값입니다. $\tau$가 작을수록 hard negative에 집중하여 더 discriminative한 표현을 학습하며, 0.1은 실험적으로 검증된 값입니다.

Q6. 이 두 실험의 결과로 논문에 어떻게 기여할 수 있는가?

Ablation study로 "공정성 정렬 방향(어느 성별을 변화시킬지)의 영향"을 분석할 수 있습니다. 결과에 따라 "단방향 정렬 vs 역방향 정렬"의 비교표를 제시하고, anchor 선택이 공정성-성능 tradeoff에 미치는 영향을 논의할 수 있습니다.
