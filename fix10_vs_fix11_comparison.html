02 — SupCon 원본 논문과의 비교

Supervised Contrastive Learning (Khosla et al., 2020)

SupCon 원본 수식

SupCon Loss (원본)
$$\mathcal{L}_{\text{SupCon}} = \sum_{i \in I} \frac{-1}{|P(i)|} \sum_{p \in P(i)} \log \frac{\exp(\text{sim}(z_i, z_p)/\tau)}{\sum_{a \in A(i)} \exp(\text{sim}(z_i, z_a)/\tau)}$$

SupCon 원본의 Anchor/Positive/Negative

  • Anchor: 배치 내 모든 샘플 (augmented view 포함)
  • Positive: 같은 클래스의 다른 샘플/augmented view
  • Negative: 다른 클래스의 모든 샘플
  • Weight: 모든 positive에 동일 가중치 (1/|P(i)|)

03 — Score-Weighted Contrastive Loss

fix10/fix11 공통 Contrastive Loss

두 버전 모두 동일한 contrastive loss 클래스를 사용합니다. 차이는 loss가 어떻게 조합되느냐에 있습니다.

Gaussian Kernel Weight (bin 대체)

Score-based weight
$$w(a, p) = \exp\!\left(-\frac{|s_a - s_p|^2}{2\sigma^2}\right), \quad \sigma = \text{std}(s_{\text{batch}}) \times \text{scale}$$

Contrastive Loss 수식

Score-Weighted Contrastive Loss
$$\mathcal{L}_{\text{con}} = \frac{1}{|\mathcal{A}|} \sum_{a \in \mathcal{A}} \left[-\frac{\sum_{p \in \mathcal{T}} w(a,p) \cdot z_a^\top \text{sg}(z_p)/\tau}{\sum_{p \in \mathcal{T}} w(a,p)} + \log\sum_{k \neq a} \exp(z_a^\top z_k/\tau)\right]$$

$\mathcal{A}$ = 여성(anchor), $\mathcal{T}$ = 남성(target). $\text{sg}(\cdot)$ = stop-gradient (detach). Target projection은 고정되어 gradient가 여성에만 흐릅니다.

05 — fix10 Loss 상세

Distribution-Fixed Loss (Idea 2)

Stability Loss (fix10 고유)

Male Feature Stability Loss
$$\mathcal{L}_{\text{stab}} = \frac{1}{|\mathcal{M}|}\sum_{i \in \mathcal{M}} \left\|f_i^{\text{pert}} - \text{sg}(f_i^{\text{orig}})\right\|_2^2$$
  • $f_i^{\text{orig}}$: perturbation 없이 DETR 통과 → 원본 feature (torch.no_grad)
  • $f_i^{\text{pert}}$: perturbation 적용 후 DETR 통과 → 변형 feature
  • 남성($\mathcal{M}$)에만 적용: "perturbation이 남성 feature를 변화시키면 패널티"
  • DETR forward 2회 필요 (원본 1회 + perturbed 1회)
# fix10: 원본 feature 추출 (no_grad) with torch.no_grad(): _, feat_orig = detr.forward_with_features(samples) # perturbed feature delta = generator(tensors) perturbed = clamp_normalized(tensors + delta) _, feat_pert = detr.forward_with_features(perturbed) # 남성 feature만 MSE loss_stability = F.mse_loss( feat_pert[male_idx].mean(dim=1), feat_orig[male_idx].mean(dim=1) )
06 — fix11 Loss 상세

Gender-Conditional Loss (Idea 1)

성별별 Detection Loss 분리 (fix11 고유)

Gender-Conditional Structure
$$\mathcal{L}_{\text{male}} = \beta_m \cdot \mathcal{L}_{\text{det}}^{\text{male}} \quad\quad \mathcal{L}_{\text{female}} = \lambda_{\text{con}} \cdot \mathcal{L}_{\text{con}} + \beta \cdot \mathcal{L}_{\text{det}}^{\text{female}}$$
  • 남성: Detection loss 만 → Generator에게 "남성을 변화시킬 이유 없음" → δ_male ≈ 0
  • 여성: Contrastive + Detection → Generator가 적극 조정
  • DETR forward 1회 (기존과 동일, 추가 비용 없음)
  • Stability Loss 없음 — 신호 부재가 곧 고정 메커니즘
# fix11: detection loss를 성별별 분리 (aux_outputs 포함) male_outputs = {k: v[male_idx] for k, v in outputs.items() if isinstance(v, Tensor)} if "aux_outputs" in outputs: male_outputs["aux_outputs"] = [{k: v[male_idx] for k, v in a.items()} for a in outputs["aux_outputs"]] loss_det_male, _ = detr.detection_loss(male_outputs, male_targets) female_outputs = {k: v[female_idx] for k, v in outputs.items() if isinstance(v, Tensor)} if "aux_outputs" in outputs: female_outputs["aux_outputs"] = [{k: v[female_idx] for k, v in a.items()} for a in outputs["aux_outputs"]] loss_det_female, _ = detr.detection_loss(female_outputs, female_targets) # 남성: det만, 여성: contrastive + det total = λ_con * loss_contrastive + β * loss_det_female + β_m * loss_det_male

11 — Q&A

예상 질문

Q1. fix10의 Stability Loss가 MSE인 이유? 다른 metric은?

MSE는 가장 단순하고 gradient가 안정적입니다. L1, cosine distance 등도 가능하지만, MSE가 "feature 벡터의 유클리드 거리 보존"이라는 의미가 가장 직관적이며, 추후 ablation 대상입니다.

Q2. fix11에서 남성이 정말 안 변하나? Det loss로도 gradient가 흐르는데?

Det loss는 "검출 결과를 유지하라"는 보수적 신호입니다. Generator 입장에서 남성 perturbation의 최적 해는 δ≈0입니다. 변화시킬 이유(contrastive)가 없으므로 자연스럽게 최소 변화. 다만 fix10의 MSE penalty만큼 강력한 보장은 아닙니다.

Q3. SupCon 원본은 augmentation이 필수인데, 우리는 왜 안 쓰나?

SupCon 원본은 같은 이미지의 2개 augmented view로 positive pair를 만듭니다. 우리는 cross-gender pair가 positive이므로 augmentation이 불필요합니다. 또한 Generator의 perturbation 자체가 일종의 learned augmentation 역할을 합니다.

Q4. 두 실험 중 어떤 걸 먼저 돌려야 하나?

fix11을 먼저 권장합니다. 더 단순하고 추가 비용이 없어 baseline으로 적합합니다. fix11이 충분한 성능을 보이면 그것으로 채택하고, 남성 feature 변화가 크다면 fix10의 stability loss를 추가하는 전략이 효율적입니다.

Q5. Gaussian kernel의 bandwidth_scale을 어떻게 정하나?

기본값 1.0은 "배치 score 분포의 1 표준편차 내 pair를 strong positive로 취급"합니다. 0.5(엄격)~2.0(관대) 범위에서 실험하며, detection score 분포가 좁으면(0.7~0.9) 작은 값이, 넓으면 큰 값이 적합합니다.
